发表于 2022-07-10 更新于 2022-07-17 分类于 github

本文解决的是如何使用github的方法

无法push文件到gitHub

本地git push推送远程分支，提示如下错误信息
zzg@LAPTOP-8R0KHL88 MINGW64 /e/idea_workspace (master)
$ git push
ssh: Could not resolve hostname github.com: No address associated with hostname
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.

先看一看重新生成SSH Key值，可以解决此问题不

git config user.name
git config user.email

ssh-keygen -t rsa -C "lvxl0416@163.com"

执行以上指令完成本机SSH Key值生成，并将生成的Key 拷贝至Git 账户/Setting/SSH key 设置。

一、github如何更新已经上传了的文件

按照下列的指令来

git add .
git commit -m "这是注释内容"
git pull origin main #main在这里表示的是github的branch
git push -u origin main

二、github如何上传文件

按照下列的指令来

git init
git add .
git branch main -M
git remote add origin git@github.com:yuanjun0416/deep_learning_processing.git
git commit -m "注释内容（此处写本次干了什么事）"
git push -u origin main

三、clone github上仓库中的某个子文件夹

现在有一个ssh:git@github.com:yuanjun0416/deep-learning-for-image-processing.git
仓库中有一个子文件夹pytorch_object_detection/faster_rcnn
形似：deep-learning-for-image-processing/pytorch_object_detection/faster_rcnn
按照下列的指令来

git init //初始化
git config core.sparsecheckout true //设置允许克隆子目录 
git 'pytorch_object_detection/faster_rcnn*' >> .git/info/sparse-checkout //设置要克隆的仓库的子目录路径
git remote add origin git@github.com:yuanjun0416/deep_learning_processing.git  //仓库名称
git pull origin master //branch为master就是master,为main就是main
